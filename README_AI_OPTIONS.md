# AI Content Generation Framework

**Standardized AI Options for Cross Tools**

This document describes the standardized AI content generation framework used across Cross tools. These options allow you to leverage AI to automatically generate titles, captions, summaries, and comprehensive analyses of your data.

---

## Overview

Cross is an AI-oriented project that generates, analyzes, and fact-checks content. The AI content generation framework provides a consistent way to create various types of AI-generated content across all `st-*` tools.

### Five Standard Content Types

| Option | Length | Output | Purpose |
|--------|--------|--------|---------|
| `--ai-title` | max 10 words | stdout | Short, punchy title |
| `--ai-short` | max 80 words | stdout | Brief caption |
| `--ai-caption` | 100-160 words | stdout | Detailed caption |
| `--ai-summary` | 120-200 words | stdout | Concise summary |
| `--ai-story` | 800-1200 words | JSON file | Comprehensive analysis |

---

## Usage

### Basic Pattern

```bash
st-<tool> --ai-<type> [--ai <provider>] <file.json>
```

**Examples:**

```bash
# Generate a short title using the default AI (xai)
st-speed --ai-title report.json

# Generate a detailed caption using Gemini
st-speed --ai-caption --ai gemini report.json

# Create a comprehensive story and save it to the JSON container
st-speed --ai-story --ai anthropic report.json
```

### Selecting the AI Provider

Use the `--ai` option to choose which AI generates the content:

- `--ai xai` (default) — X.AI's Grok
- `--ai gemini` — Google's Gemini
- `--ai anthropic` — Anthropic's Claude
- `--ai openai` — OpenAI's GPT
- `--ai perplexity` — Perplexity AI

**Example:**

```bash
st-speed --ai-summary --ai gemini performance.json
```

**Important Note About Filtering:**

The `--ai` flag has two effects:

1. **Filters display tables** — Only shows that AI's performance
2. **Selects content generator** — That AI writes the analysis

However, **AI analysis always receives the full dataset** (all AIs), even when display is filtered. This ensures the AI can make meaningful comparisons and provide context.

**Example:**
```bash
# Display shows only Gemini, but Gemini's analysis compares all AIs
st-speed --ai-title --ai gemini report.json
# Output: "OpenAI 7x Faster Than Anthropic Fact-Checking"
#         (Gemini analyzed all AIs and made this comparison)
```

This design provides the best of both worlds:
- Focused display (what you're interested in)
- Comprehensive analysis (proper context and comparisons)

## Cache Control

### Default Behavior: Cache Enabled

AI content generation uses caching by default for speed and cost efficiency:

```bash
# Uses cached response if available (fast, free)
st-speed --ai-title report.json
```

### Force Fresh Content: --no-cache

Generate fresh content (bypass cache):

```bash
# Always calls AI (slower, costs API credits)
st-speed --ai-title --no-cache report.json
```

### When to Use --no-cache

1. **Data has changed** — New timing data added
2. **Debugging** — Testing AI behavior
3. **Poor cached content** — Want to try again
4. **Prompt changes** — Modified prompts in code

### Verbose Mode with Cache

See cache status and generation details:

```bash
st-speed --ai-title --no-cache --verbose report.json
```

**Output:**
```

  Calling xai to generate title...
  Prompt length: 950 chars
  Cache: disabled
  Generated 7 words
  Preview: OpenAI 8x Faster Than Anthropic...
OpenAI 8x Faster Than Anthropic in Fact-Checking
  (7 words)
──────────────────────────────────────────────────────────────────────
```

### 1. `--ai-title` — Short Title

**Purpose:** Generate a concise, data-driven title that captures the key finding.

**Characteristics:**
- Maximum 10 words
- Punchy and specific
- Mentions AI names or key metrics when relevant
- No unnecessary articles (a, an, the)

**Output:** Plain text to stdout

**Example Output:**
```
OpenAI Leads Speed by 3x Over Anthropic
```

**When to Use:**
- Creating headlines for reports
- Generating social media posts
- Summarizing findings quickly

---

### 2. `--ai-short` — Short Caption

**Purpose:** Provide a brief, focused caption with key findings.

**Characteristics:**
- Maximum 80 words
- Single paragraph
- States fastest and slowest performers with numbers
- Includes one key insight (variance, consistency, recommendation)

**Output:** Plain text to stdout

**Example Output:**
```
OpenAI leads fact-checking at 34s average (range 3-64s), followed by 
Perplexity at 49s. Anthropic trails at 4m23s average. OpenAI and 
Perplexity show low variance (StdDev ~25s), enabling predictable 
sub-minute performance, while Anthropic's 228s StdDev risks timeouts 
in production. Default to OpenAI or Perplexity for reliability.
```

**When to Use:**
- Adding context to data visualizations
- Email notifications with performance data
- Quick summaries for stakeholders

---

### 3. `--ai-caption` — Detailed Caption

**Purpose:** Provide a comprehensive caption with both obvious and non-obvious insights.

**Characteristics:**
- 100-160 words total
- 1-2 paragraphs
- Highlights performance ranges and patterns
- Points out non-obvious insights (variance, outliers, consistency)
- Data-driven with specific numbers

**Output:** Plain text to stdout

**Example Output:**
```
OpenAI leads in fact-checking speed with an average of 34 seconds 
per job (median 35s, range 3-64s), followed closely by Perplexity 
at 49 seconds. Gemini averages 87 seconds, xAI 109 seconds, and 
Anthropic trails at 263 seconds.

A key non-obvious pattern is consistency: OpenAI and Perplexity 
show low variance (StdDev 28.4s and 25.4s), enabling predictable 
sub-minute performance, while slower models suffer from high 
variability—Anthropic's 228.8s StdDev and xAI's 107.9s (pulled 
up by a 254s outlier) risk timeouts in production. For production, 
default to OpenAI or Perplexity to balance speed and reliability.
```

**When to Use:**
- Documenting performance analyses
- Creating README sections
- Detailed reports for technical audiences

---

### 4. `--ai-summary` — Concise Summary

**Purpose:** Provide a thorough but concise analysis suitable for documentation.

**Characteristics:**
- 120-200 words total
- 2-3 paragraphs
- Covers both speed and consistency
- Explains implications for production use
- Identifies best use cases for each AI

**Output:** Plain text to stdout

**Example Output:**
```
Performance analysis reveals distinct speed and consistency patterns 
across five AI providers. OpenAI leads at 34s average, followed by 
Perplexity (49s), Gemini (87s), xAI (109s), and Anthropic (263s). 
Each sample represents a complete fact-check job involving 18-19 
AI calls per segment.

Consistency is the hidden differentiator. OpenAI and Perplexity 
maintain low variance (StdDev ~25s), delivering predictable 
sub-minute performance. In contrast, Anthropic's 228s StdDev 
and xAI's 107s variance create unpredictable latency—xAI's 
254s outlier pushes the max beyond 4 minutes despite an 88s median.

For production systems requiring reliability, OpenAI and Perplexity 
are clear choices. Reserve Gemini, xAI, and Anthropic for latency-
tolerant workflows, and implement timeouts beyond their maximum 
observed times (e.g., 5 minutes for Anthropic, 3 minutes for xAI).
```

**When to Use:**
- Technical documentation
- Architecture decision records
- Performance review documents

## Implementation Details

### Word Count Validation

All content types enforce strict word count requirements:

```python
requirements = {
    "title": (1, 10),       # max 10 words
    "short": (1, 80),       # max 80 words
    "caption": (100, 160),  # 100-160 words
    "summary": (120, 200),  # 120-200 words
    "story": (800, 1200),   # 800-1200 words
}
```

If content falls outside these ranges, a warning is displayed but the content is still returned (allowing human judgment).

### Data-Driven Validation

Content (except titles) must include numerical data to be considered valid. This ensures AI-generated content is grounded in the actual performance data.

### Caching

All AI calls use the standard Cross caching framework (`api_cache`). Repeated requests with the same data will return cached results, saving time and API costs.

### Error Handling

If content generation fails:
1. An error message is displayed
2. Suggestions are provided (try different AI provider)
3. In verbose mode (`-v`), full traceback is shown
4. The tool continues (doesn't crash)

## Best Practices

### 1. Choose the Right Content Type

- **Quick updates?** → `--ai-short`
- **Documentation?** → `--ai-caption` or `--ai-summary`
- **In-depth analysis?** → `--ai-story`
- **Social media?** → `--ai-title`

### 2. Select Appropriate AI

Different AIs have different strengths:

- **Gemini** — Fast, concise, good for titles and short captions
- **Anthropic** — Detailed, analytical, good for summaries and stories
- **OpenAI** — Balanced, reliable for all content types
- **XAI** — Fast, direct, good for quick insights
- **Perplexity** — Data-focused, good for technical content

### 3. Use Verbose Mode for Debugging

```bash
st-speed --ai-caption --verbose report.json
```

Shows:
- Prompt length
- AI being called
- Word count generated
- Validation results
- Preview of content

### 4. Iterate on Content

Since caching is enabled, you can try different content types quickly:

```bash
st-speed --ai-short report.json
st-speed --ai-caption report.json
st-speed --ai-summary report.json
```

The AI will reuse cached data, so these run almost instantly.

### Implementation Pattern

To add this framework to a tool:

1. **Import functions:**
   ```python
   from ai_handler import process_prompt, get_content, get_default_ai
   ```

2. **Add argument group:**
   ```python
   ai_group = parser.add_argument_group('AI Content Generation')
   ai_group.add_argument('--ai-title', action='store_true',
                       help='Generate short title (max 10 words) → stdout')
   # ... add other options
   ```

3. **Call generation function:**
   ```python
   content = generate_ai_content(
       data_summary, 
       content_ai, 
       content_type, 
       verbose
   )
   ```

4. **Handle output:**
   ```python
   if content_type == "story":
       save_story_to_container(...)
   else:
       print(content)  # to stdout
   ```

## Troubleshooting

### "No title generated" or "Empty content"

**Problem:** Title section appears but no text is shown

**Causes:**
1. **Bad cached response** — Previous bug cached empty content
2. **API error** — AI call failed silently
3. **Extraction error** — Content not properly extracted from response

**Solutions:**

1. **Force fresh call with verbose:**
   ```bash
   st-speed --ai-title --no-cache --verbose report.json
   ```
   
   Look for:
   - "Generated X words" — If 0, AI returned nothing
   - "Content generation failed" — API error
   - Error messages about authentication

2. **Try different AI:**
   ```bash
   # If xai fails, try gemini
   st-speed --ai-title --ai gemini --no-cache report.json
   
   # Or anthropic
   st-speed --ai-title --ai anthropic --no-cache report.json
   ```

3. **Check data exists:**
   ```bash
   # Verify timing data in file
   st-speed report.json
   # Should show performance tables
   ```

4. **Check API keys:**
   ```bash
   # Verify API key is set
   echo $XAI_API_KEY | cut -c1-10
   # Should show: xai-XXXXXX
   
   # Or check .env file
   grep XAI_API_KEY /Users/Matt/github/cross/.env
   ```

### "Content generation failed: authentication error"

**Problem:** AI API key not found

**Solution:**
```bash
# Check .env file in project root
cat .env | grep API_KEY

# Ensure correct variable name:
#   XAI_API_KEY
#   GEMINI_API_KEY
#   ANTHROPIC_API_KEY
#   OPENAI_API_KEY
#   PERPLEXITY_API_KEY
```

### "Warning: title too long (15 words, max 10)"

**Problem:** AI generated content exceeding word limit

**Solution:**
- Content is still displayed (warning only)
- Try a different AI: `--ai gemini` (often more concise)
- File an issue if persistent

### "Caption lacks data (no numbers found)"

**Problem:** AI generated generic text without data

**Solution:**
- Re-run command (AI responses can vary)
- Try a different AI: `--ai anthropic` (more analytical)
- Ensure your data file contains valid timing data
