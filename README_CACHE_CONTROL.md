# Cache Control for AI Content Generation

**Managing API Response Caching in Cross Tools**

---

## Overview

All Cross tools that make AI API calls support caching to:
- **Save time** — Cached responses return instantly
- **Save money** — No repeated API charges
- **Enable iteration** — Quickly try different options

However, sometimes you need **fresh content** (especially when debugging or when data has changed).

---

## Cache Options

### Default Behavior

**AI content generation uses cache by default:**

```bash
st-speed --ai-title report.json
# Uses cached response if available (fast, free)
```

### Disable Cache

Force fresh AI calls with `--no-cache`:

```bash
st-speed --ai-title --no-cache report.json
# Always calls AI (slower, costs API credits)
```

### Enable Cache Explicitly

Use `--cache` to be explicit (same as default):

```bash
st-speed --ai-title --cache report.json
# Uses cached response if available (explicit)
```

**Note:** `--no-cache` overrides `--cache` if both are specified.

---

## When to Use --no-cache

### 1. **Data Has Changed**

If you've added new timing data:

```bash
# Added new fact-check data
st-fact --ai all -s 5 report.json

# Generate fresh analysis
st-speed --ai-caption --no-cache report.json
```

### 2. **Debugging AI Output**

When testing or debugging:

```bash
# See what AI generates without cache
st-speed --ai-title --no-cache --verbose report.json
```

### 3. **Unsatisfied with Cached Content**

If cached content is poor quality:

```bash
# Try different AI with fresh call
st-speed --ai-caption --ai anthropic --no-cache report.json
```

### 4. **Testing Different Prompts**

When you've modified prompts in the code:

```bash
# Force fresh AI call with new prompt
st-speed --ai-summary --no-cache report.json
```

### 5. **Suspected Cache Corruption**

If you suspect bad data in cache:

```bash
# Clear cache for this specific request
st-speed --ai-title --no-cache report.json
```

---

## When to Use --cache (default)

### 1. **Iterating on Output**

When trying different content types:

```bash
# All of these will reuse cached data (very fast)
st-speed --ai-title report.json
st-speed --ai-short report.json
st-speed --ai-caption report.json
```

**Note:** Different content types use different prompts, so they won't share cache. But multiple runs of the SAME content type will cache.

### 2. **Production Workflows**

When reliability and cost matter:

```bash
# Reuse cached responses in scripts
for file in *.json; do
    st-speed --ai-caption "$file" >> output.txt
done
```

### 3. **Demonstrating Tools**

When showing tools to others:

```bash
# Fast demo (no API delays)
st-speed --ai-title demo.json
```

## Cache Behavior Details

### What Gets Cached?

Cache key includes:
- AI provider (xai, gemini, anthropic, openai, perplexity)
- Prompt text (including data tables)
- Model parameters

**Same data + same AI + same content type = cache hit**

### What Doesn't Get Cached?

Cache miss when:
- Different AI provider (`--ai gemini` vs `--ai openai`)
- Different content type (`--ai-title` vs `--ai-short`)
- Different data (timing data changed)
- Cache disabled (`--no-cache`)

### Cache Location

Managed by `ai_handler.py` framework:
- Stored in `api_cache` directory (implementation detail)
- Persists across sessions
- Shared across all `st-*` tools

## Troubleshooting

### "I'm getting the same response every time"

**Problem:** Cached response being reused

**Solution:**
```bash
# Force fresh AI call
st-speed --ai-title --no-cache report.json
```

### "Content seems outdated"

**Problem:** Data changed but cache not invalidated

**Solution:**
```bash
# Use --no-cache when data has changed
st-speed --ai-caption --no-cache report.json
```

### "Empty or incorrect title"

**Problem:** Bad response got cached

**Solutions:**

1. **Use --no-cache:**
   ```bash
   st-speed --ai-title --no-cache report.json
   ```

2. **Try different AI:**
   ```bash
   st-speed --ai-title --ai anthropic --no-cache report.json
   ```

3. **Use verbose mode to debug:**
   ```bash
   st-speed --ai-title --no-cache --verbose report.json
   ```

## Technical Notes

### Cache Implementation

From `ai_handler.py`:

```python
def process_prompt(ai_make, prompt, verbose=False, use_cache=True):
    """
    Process a prompt through the specified AI provider.
    
    Args:
        ai_make: AI provider (xai, gemini, anthropic, openai, perplexity)
        prompt: Prompt text
        verbose: Show debugging output
        use_cache: Use cached responses if available
    
    Returns:
        Tuple: (gen_payload, client, response, ai_model)
    """
    # Check cache first
    if use_cache:
        cached = get_cached_response(...)
        if cached:
            return cached
    
    # Call AI
    response = call_ai(...)
    
    # Store in cache
    if use_cache:
        store_cached_response(...)
    
    return response
```

### Cache Key Generation

Cache key based on:
- AI provider
- Model name
- Prompt text (full text, including data)
- System parameters

**Important:** Changing ANY of these invalidates cache.

## Diagnosing Cache Issues

### Issue: "Getting same response with different data"

**Diagnosis:**
```bash
# Check if data actually changed
st-cat report.json | jq '.story[-1].timing'

# Force fresh call
st-speed --ai-title --no-cache report.json
```

### Issue: "No title generated"

**Diagnosis:**
```bash
# Verbose mode shows what's happening
st-speed --ai-title --no-cache --verbose report.json

# Check for errors:
# - "Content generation failed" → API error
# - "Empty content" → AI returned nothing
# - "too short" → Validation failed
```

### Issue: "Title is empty string"

**Diagnosis:**
```bash
# Full verbose output
st-speed --ai-title --no-cache -v report.json 2>&1 | cat -A

# Look for:
# - Empty lines between delimiters
# - Error messages
# - "Debug: content=''" messages
```
